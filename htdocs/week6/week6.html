HW0:

1. What are JavaScript browser sniffers?

Answer:

  Browser sniffing is the act of detecting the web browser a visitor is using in order to serve version-specific pages, scripts, images, or other content. JavaScript Browser Sniffer is a browser identifier written in JavaScript. It will tell which browser, version and operating system the visitor is using. It also detects bots and the Flash plugins.

2. Why are they so commonly used in JavaScript?

Answer:

  The technique was popular around 10 years ago because Microsoft and Netscape introduced their own technologies and web standards were in their infancy. And technically, browser sniffing appears to be relatively straightforward. We can simply obtain a user agent string using navigator.userAgent in JavaScript. All of the listing reasons makes browser sniffing commonly used in JavaScript.
  However, I also find some materials which argue that browser sniffing shouldn’t be used anymore. 5 reasons are listed to refute sniffers.
  1. String parsing is tough;
  2. There are many browsers to handle;
  3. Reduced scalability and increased maintenance;
  4. Browser masquerading;
  5. It’s rarely required;
  So I think we may have to consider it carefully when we are about to use sniffers. If only a few browsers have to be supported, the sniffer may work well. But we have to take tradeoffs when a wide variety of platforms require support.
