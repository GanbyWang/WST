Q1: Which category so-called "Netscape plug-in" technology belongs to? Please describe the basic idea of the technology, and give your reasoning on its position in the technology categories.

Answer: I think “Netscape plug-in” belongs to distributed object technologies.
Basic idea of “Netscape plug-in”:
  The name, “Netscape plug-in”, obviously indicates that the technology focuses on developing plug-ins, and in this case, for both browser side and server side. The APIs it defines are known as NPAPI (Netscape Plug-in Application Programming Interface) and NSAPI (Netscape Server Application Programming Interface), and the former is for browsers while the latter is for servers.
  In NPAPI architecture, a plugin declares content types it can handle. When the browser encounters a content type it cannot handle natively, it loads the appropriate plugin, sets aside space within the browser context for the plugin to render and then streams data to it. The plugin is responsible for rendering the data. The plugin runs in-place within the page, as opposed to older browsers that had to launch an external application to handle unknown content types.
  Each plug-in that use NSAPI implements one or more Server Application Functions (SAFs). To use a SAF, an administrator must first configure the server to load the plug-in that implements that SAF. This is typically controlled by a configuration file named magnus.conf. Once the plug-in is loaded, the administrator can configure when the server should invoke the SAF and what parameters it should be passed. This is typically controlled by a configuration file named obj.conf.

Reasons:
  The most noteworthy characteristic of distributed objects technology is that the objects involved in the system are distributed across different address spaces but work together. The “Netscape Plug-in” technology enables the system to distribute mixed objects over servers and browsers, which coincides with the definition of distributed objects technology. What’s more, CORBA and ActiveX/DCOM are typical distributed objects technology as we know, while I’ve learned that ActiveX is identical to NPAPI, so I suppose “Netscape Plug-in” technology is also a distributed objects technology.

Q2: Think about the statusless character of World Wide Web. Give some good points and bad points of it. Can a web server know exactly how many browsers are reading its document?

Answer:
Characteristics of the Statusless WWW
Good points of the “statusless” character Bad points of the “statusless” character
  1. A client won’t occupy the web resource for a long time, which improves the utilization of servers’ resources and accelerates the communication.

  2. A server won’t waste the time making sure the document is received completely and can start responding the next request at once, which shortens the client’s waiting time and more clients are responded.

  3. This solution is simple and straight-forward, so it’s easy to build a system working on such a protocol and maintain the flexibility of the protocol.
  1. It’s rather possible to receive an incomplete document when the network doesn’t work well due to the lack of reliability.

  2. With the developments of web, sometimes we need to record the state of communications, but the server doesn’t have the ability.

Conclusion: Since the protocol doesn’t keep a record of requests and responses, it’s impossible to count how many browsers are reading its documents.
